What is a smart contract?

A smart contract is deterministic code stored on a blockchain. Anyone can verify its logic; transactions invoke functions and change state according to the rules you encoded—no middle party required.

Immutable by default

Deployed bytecode is fixed unless you built in upgrade patterns (proxies). Plan migrations and versioning up front.

Gas & execution limits

Every opcode costs gas. Loops, storage writes, and external calls dominate bills—profile before shipping.

Trust boundaries

Contracts interact with EOAs, other contracts, or oracles. Assume malicious callers and untrusted inputs.

Lifecycle

A disciplined pipeline catches bugs early—cheaper than an incident on chain.

Specify Threat model, roles, invariants, and failure modes (pause, upgrade, migration).
Implement Solidity with checks-effects-interactions, custom errors, and minimal external calls.
Verify Unit tests, fuzzing, static analysis, formal specs where justified.
Deploy Scripted deployments, verified sources, monitored events, and incident runbooks.

Common patterns

Building blocks you will recognize across production systems—know them before you customize.

ERC-20 token

Fungible balances with transfer, approve, and allowance mechanics.

interface IERC20 { function transfer(address to, uint256 v) external returns (bool); }

Access control

Role-based guards (onlyRole), Ownable patterns, or timelocked admins—never leave privileged calls wide open.

Pull over push

Let users withdraw funds instead of pushing Ether to arbitrary addresses—avoids reentrancy and griefing.

Security checklist

Non-exhaustive reminders—pair with professional review for anything holding real assets.

Reentrancy

Use checks-effects-interactions; consider reentrancy guards for external calls.

Integer math

Solidity 0.8+ has built-in overflow checks; still watch casting and assembly.

Oracle risk

Spot prices can be manipulated—use TWAPs, multiple sources, or circuit breakers.
